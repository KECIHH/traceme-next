# 当前工作台验收基线

本文件是当前主线验收基线，供后续窗口按统一标准验证已经实现的 personal beta 工作台。

## 范围

- 当前验收范围：personal beta workbench。
- 本清单只验收当前主线：生成、对比、导出、登录/保存、历史、详情、版本恢复、分享、公开只读、软删除、最近删除、恢复删除。
- 使用可丢弃测试行程和占位描述，不记录真实账号、真实环境、真实分享链接或任何敏感值。
- 当前事实提醒：delete / recently deleted / restore-deleted UI 已实现，不作为缺口规划。

## 自动验证命令

自动验证命令：

```bash
npm test
npm run lint
npm run build
npx tsc --noEmit
```

- `npm run build` 后单独跑 `npx tsc --noEmit`。
- 不要并行运行 `npm run build` 和 `npx tsc --noEmit`。
- 记录每条命令结果：`通过`、`失败 + 简要原因`，或 `未验证 + 原因`。

## 浏览器验收清单

浏览器验收清单：每项只能记录 `通过`、`失败 + 简要原因`，或 `未验证 + 原因`。没有实际检查的项目不得写成通过。

| 项目 | 验收路径 |
| --- | --- |
| 生成 | 从 `/` 提交有效旅行请求，确认页面展示结构化 `TripPlan`：总览、每日行程、景点、餐饮、住宿、交通、预算、准备清单、风险提醒、用户自行确认事项和免责声明。 |
| 对比 | 主方案生成后执行对比，确认展示变体、节奏建议、预算风险、行程冲突、替换建议、人工确认事项和免责声明，且口径仍为规划参考。 |
| 复制 | 点击复制全文，确认自动复制成功；若浏览器不允许自动复制，确认手动复制区域可见且内容非空。复制内容应包含每日行程、用户自行确认事项和免责声明。 |
| Markdown 下载 | 点击 Markdown 下载。若环境能拿到下载文件，确认 `.md` 非空并包含每日行程、用户自行确认事项和免责声明；若环境不支持下载落盘检查，记录 `未验证 + 原因`，并单独记录页面是否出现下载触发反馈。 |
| 浏览器打印/保存 PDF入口 | 点击浏览器打印/保存 PDF 入口，确认页面触发浏览器打印反馈。系统打印弹窗可能无法由自动化环境检查；这不是 server-side precision PDF。 |
| 登录态 | 在 Auth/database 可用时，确认登录态显示安全，不暴露 owner email、真实账号、session token 或 secret。若环境不具备登录/数据库，记录 `未验证 + 原因`。 |
| 手动保存 | 对已生成方案执行手动保存，确认保存成功，并能进入 `/trips` 或 `/trips/[id]`。生成结果不得被视为自动保存。 |
| `/trips` | 打开 `/trips`，确认只列出当前用户未删除行程，已删除行程不可见；未登录、空列表和错误状态均为安全文案。 |
| `/trips/[id]` | 打开保存详情页，确认当前快照、导出入口、版本历史、分享控制和删除控制只在 owner 详情页出现。 |
| 版本恢复 | 在至少两个版本的测试行程上恢复旧版本，确认恢复会创建新的当前版本，不覆盖旧版本历史。 |
| 分享创建 | 在 owner 详情页创建分享链接，确认完整链接只出现一次且可复制或手动复制；验收记录不得保存 raw share token 或真实完整 URL。 |
| 分享复制 | 确认分享链接复制成功反馈；若剪贴板不可用，确认手动复制 fallback 可用。 |
| 分享撤销 | 撤销活跃分享链接，确认 owner UI 标记为已撤销，且该链接不能再次撤销。 |
| 公开只读页 | 在无登录上下文打开 `/shared/trips/[token]`，确认展示固定版本行程快照，且没有保存、恢复、撤销、删除、编辑、owner 管理或 debug 控制。 |
| 公开不可用 | 分享撤销后打开旧链接，确认显示统一不可用状态，不泄露链接是不存在、撤销、过期、被删除还是跨用户。 |
| 删除 | owner 删除保存行程时必须有确认；删除后该行程离开普通 `/trips`，普通 `/trips/[id]` 详情不可用。 |
| `/trips/deleted` | 打开 `/trips/deleted`，确认最近删除列表能看到该测试行程，并只提供 owner 可用的安全恢复控制。 |
| 恢复删除 | 恢复最近删除行程，确认该行程回到 `/trips`，保存详情和历史可再次访问。 |
| 旧分享链接恢复后仍不可用 | 删除导致旧分享不可用后，即使恢复删除，该旧分享链接仍不可用；需要 owner 新建分享链接才可公开访问。 |

## 安全检查

安全检查：不记录真实账号、域名、IP、DATABASE_URL、AUTH_SECRET、OAuth secret、AI key、session token、bearer/header、raw share token、token hash、owner email、SQL detail、stack trace。

可使用占位符：

- `[本地验证 URL]`
- `[测试账号]`
- `[测试数据库]`
- `[测试行程]`
- `[测试分享链接]`
- `[真实 AI Provider]`

## 环境限制规则

环境限制规则：能验证就记录通过；环境不具备就记录“未验证 + 原因”；不得伪造通过。

- 浏览器下载、剪贴板、系统打印弹窗、真实登录、测试数据库、真实 AI Provider 等能力不可用时，应明确写出限制。
- 既有单元测试、源码阅读或历史记录不能替代本轮浏览器验收结果。
- 如果命令或浏览器项目失败，只记录失败和简要原因；除非当前轮次明确要求修复，否则不得扩大为源码修复。

## 非目标

非目标：不验收 admin、production、maps、weather、search、hard delete、live booking、payments、server-side precision PDF。
